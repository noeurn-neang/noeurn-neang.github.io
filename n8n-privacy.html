Privacy Policy for Neang Noeurn's n8n Projects

Last Updated: October 25, 2025

Welcome to the privacy policy for my automation projects built with n8n, developed by Neang Noeurn. This policy outlines how data is handled when my n8n workflows interact with Google APIs or other services to automate tasks, such as trend monitoring, data analysis, or other functionalities.

1. Data We Collect

My n8n workflows, including projects like Trend Watcher and others, are designed to automate tasks using APIs, such as Google APIs (e.g., Google Trends, Google Sheets). These workflows typically collect:

  • Anonymized, non-personal data: Aggregated data from APIs (e.g., trend statistics, public datasets).
  • Configuration data: Settings or inputs provided to the n8n workflow to perform its tasks (e.g., search terms, API keys).
  • No personal user data: Unless explicitly designed for a specific project, my workflows do not collect, store, or process personal information from end users.

If a project requires personal data (e.g., for authenticated API access), this will be clearly disclosed in the project’s documentation.

2. How We Use Data

Data accessed by my n8n workflows is used solely to:

  • Execute automation tasks (e.g., fetching trends, processing data, generating reports).
  • Deliver outputs to configured destinations (e.g., email, Slack, databases).
  • Improve workflow functionality and debug issues.

All data processing occurs within the n8n platform or connected services as specified by the workflow’s design.

3. Data Sharing and Disclosure

We do not share, sell, or disclose any data accessed via Google APIs or other services with third parties, except:

  • As required to comply with applicable laws or regulations.
  • With service providers (e.g., n8n cloud, API providers) necessary for the workflow’s operation, under strict compliance with their terms.

Data remains confined to the scope of each project’s n8n workflow and is not transferred beyond its intended use.

4. Google API Compliance

Our use and transfer of information received from Google APIs to any other app will adhere to the Google API Services User Data Policy, including the Limited Use requirements. We ensure all API interactions are secure and compliant with Google’s terms.

5. Data Security

We implement reasonable measures to secure n8n workflows, including:

  • Secure API authentication (e.g., OAuth 2.0, API keys).
  • Encrypted connections for data transfers.
  • Regular updates to workflows to address security vulnerabilities.

Since most workflows handle non-personal data, the risk of data breaches is minimal. If personal data is used, additional safeguards will be applied and disclosed.

6. Changes to This Policy

We may update this privacy policy to reflect changes in our projects, APIs, or legal requirements. The latest version will always be available at this URL.
